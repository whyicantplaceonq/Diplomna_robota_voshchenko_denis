# deployment.md — Розгортання Tycoon Simulator у production

> **Production** для UEFN-проекту = публікація острова в Fortnite Creative,
> де він стає доступним для мільйонів гравців по всьому світу.
> Роль "сервера" виконує інфраструктура Epic Games.

### Компоненти системи

| Компонент | Тип | Хто підтримує |
|---|---|---|
| Verse-код | Ігрова логіка | Розробник |
| Island Assets | 3D-об'єкти, мапи | Розробник |
| Game Server | Multiplayer інфраструктура | Epic Games (AWS) |
| БД гравців | Стан, інвентар, валюта | Epic Games |
| CDN/Storage | Зберігання острова | Epic Games |
| Fortnite Client | Рендеринг, мережа | Epic Games |

> **Важливо:** UEFN-проект не вимагає власного сервера, БД або хостингу.
> Вся серверна інфраструктура керується Epic Games.

## 3. Покрокове розгортання (публікація острова)

### 3.1 Підготовка до публікації

```
1. Переконайтесь що всі зміни закомічені:
   git status  →  "nothing to commit"

2. Запустіть статичний аналіз:
   python scripts/verse_lint.py Verse/
   →  0 errors, 0 warnings

3. Скомпілюйте Verse:
   UEFN → Verse → Build Verse Code (Ctrl+Shift+B)
   →  "Build successful"

4. Протестуйте острів:
   UEFN → Launch Session (Ctrl+F8)
   →  перевірте ключові механіки
```

### 3.2 Налаштування метаданих острова

```
UEFN → Island Settings → заповніть:
  ├── Island Name:        "Tycoon Simulator"
  ├── Description:        (до 200 символів)
  ├── Tags:               tycoon, simulator, economy
  ├── Max Players:        16
  └── Island Code:        (генерується автоматично)
```

### 3.3 Публікація

```
UEFN → Island → Publish to Fortnite

Кроки у діалозі:
  1. "Review" → перевірте метадані
  2. "Content Check" → автоматична перевірка Epic Games
  3. "Publish" → підтвердіть публікацію
  4. Дочекайтесь: "Your island has been published!"
```

### 3.4 Перевірка після публікації

```
1. Відкрийте Fortnite
2. Discover → Search → введіть Island Code
3. Запустіть острів
4. Перевірте:
   ✓ Острів завантажується без помилок
   ✓ Тригери спрацьовують
   ✓ Білборд відображає правильний текст
   ✓ Золото списується при покупці
   ✓ Об'єкти переміщуються при розблокуванні
```

## 4. Конфігурація середовища

### Структура файлів проекту

```
project_root/
├── Verse/                    # Вихідний код — комітиться в Git
│   └── unlockables_device.verse
├── Content/                  # Ассети — комітяться вибірково
│   └── Maps/
├── Saved/                    # НЕ комітити (в .gitignore)
├── Intermediate/             # НЕ комітити
└── *.digest.verse            # НЕ комітити
```

### Змінні середовища

UEFN не використовує `.env` файли. Конфігурація проекту зберігається у:
- `Island Settings` (в UEFN редакторі)
- `@editable` полях Verse класів (налаштовуються в редакторі)

### @editable параметри unlockables_device

| Параметр | Тип | Значення за замовч. | Опис |
|---|---|---|---|
| `DefaultMoveDistance` | float | 1500.0 | Висота переміщення об'єктів (units) |
| `PurchaseDatas` | []purchase_data | [] | Масив покупок |
| `GoldConditionalButton` | device | — | Пристрій перевірки золота |
| `ClaimerTrigger` | device | — | Тригер взаємодії |
| `ClaimerBillboard` | device | — | Білборд відображення |

## 5. Перевірка працездатності

### Чеклист після публікації

```
□ Island Code доступний у Fortnite Discovery
□ Острів запускається без вильотів
□ OnBegin виконується (об'єкти ховаються під землю)
□ Перший тригер активний та відображає ціну
□ Білборд показує назву та ціну першої покупки
□ При натисканні тригера — перевірка золота працює
□ При достатньому золоті — об'єкт з'являється
□ Тригер переміщується до наступної покупки
□ При виході гравця — стан скидається
```
