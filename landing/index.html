// 01 — актуальність

Актуальність теми

Ігрова індустрія є одним із найбільших секторів розваг у світі. Платформа Fortnite Creative 2.0 з появою UEFN відкрила нові можливості для розробників — повноцінний Unreal Engine безпосередньо у Fortnite.

Мова програмування Verse є принципово новим підходом до створення ігрової логіки, поєднуючи функціональні та об'єктно-орієнтовані парадигми у контексті реального часу.

Жанр тайкун-симулятора дозволяє повною мірою дослідити реалізацію економічних систем, прогресії гравця та інтерактивного UI засобами UEFN.

# Verse — мова для UEFN using { /Fortnite.com/Devices } using { /Verse.org/Simulation } tycoon_manager := class(creative_device): # Баланс гравця var Balance : int = 0 OnBegin()<suspends>: void = loop: CollectIncome() Sleep(5.0) CollectIncome(): void = set Balance += CalculateIncome()

// 02 — мета та завдання

Мета та основні завдання

Розробити повноцінний тайкун-симулятор у середовищі UEFN, дослідити можливості мови Verse для реалізації ігрових економічних систем.

ЗАВДАННЯ 01

Аналіз платформи UEFN

Дослідити архітектуру Unreal Editor for Fortnite, можливості мови Verse та інструменти розробки Creative 2.0.

ЗАВДАННЯ 02

Проєктування економіки

Розробити систему збору ресурсів, виробництва та управління балансом гравця з прогресивним масштабуванням.

ЗАВДАННЯ 03

Реалізація ігрової логіки

Імплементувати механіки будівництва, апгрейдів та системи розблокування на мові Verse.

ЗАВДАННЯ 04

Розробка інтерфейсу

Створити інтуїтивний HUD з відображенням балансу, прогресу та інтерактивними елементами управління.

ЗАВДАННЯ 05

Тестування та оптимізація

Провести ігрове тестування, виявити та усунути помилки, оптимізувати продуктивність симулятора.

// 03 — методологія

Методологія дослідження

01

Огляд літератури та платформи

Аналіз документації UEFN, специфікації мови Verse, дослідження існуючих тайкун-ігор та їх механік на платформі Fortnite Creative.

02

Проєктування архітектури

Розробка геймдизайн-документу, визначення структури модулів Verse, проєктування економічної моделі та балансу гри.

03

Ітеративна розробка

Покрокова реалізація функціоналу: від базових механік збору ресурсів до повноцінної системи будівництва та прогресії.

04

Тестування та аналіз

Проведення ігрових сесій, збір зворотного зв'язку, аналіз ігрового балансу та порівняння з аналогами у жанрі.
